Contrastive Learning for Classification
of Rare Astronomical Transients

PROJECT DEVELOPMENT FLOW · SLIT · DATA SCIENCE

▸ Research & Development Pipeline
01
STEP 01 Data Collection & Preprocessing
Multi-band light curve data (g, r, i filters) is fetched from the publicly available Zwicky Transient Facility (ZTF) via the ALeRCE API. Raw observations are cleaned, magnitude-normalized, and temporally aligned across all photometric bands to produce a consistent, high-quality dataset.
ZTF Survey ALeRCE API Light Curves Normalization Missing Value Handling
02
STEP 02 Domain-Specific Data Augmentation
To simulate realistic observational variability, domain-informed augmentation strategies are applied to generate positive and negative sample pairs for contrastive learning. These mimic real-world survey conditions such as weather gaps, cadence variation, and photometric noise.
Temporal Jittering Magnitude Scaling Noise Injection Sequence Cropping Filter Transforms
03
STEP 03 Self-Supervised Contrastive Pretraining
A feature encoder (1D CNN + GRU or Transformer) is trained using a SimCLR / BYOL contrastive framework on massive unlabelled light curve data. The NT-Xent loss pulls augmented views of the same event closer and pushes apart views from different events, building powerful latent representations without any labels.
SimCLR / BYOL 1D-CNN + GRU NT-Xent Loss Embedding Space PyTorch Unlabelled Data
04
STEP 04 Few-Shot Fine-Tuning for Classification
The pretrained encoder is extended with a classification head (dense layers + softmax) and fine-tuned on a small labelled dataset covering rare transient classes — supernovae, kilonovae, tidal disruption events (TDEs), and cataclysmic variables. The encoder can be frozen or partially unfrozen for optimal transfer.
Few-Shot Learning Softmax Head Kilonovae TDE Cataclysmic Variables Transfer Learning
05
STEP 05 Evaluation & Baseline Comparison
Performance is benchmarked against supervised baselines (CNN, RNN, LightGBM trained on labels only). Embedding quality is visualised using t-SNE / UMAP. Quantitative metrics confirm improvements in recall and F1 for rare classes where supervised methods fail due to imbalance.
Accuracy / F1 Precision / Recall t-SNE UMAP Confusion Matrix Baseline CNN/RNN
⭑
OUTPUT Deployment & Real-Time Inference
The trained pipeline is deployed as a real-time inference service. New multi-band light curves from live surveys are normalised, encoded, and classified — outputting predicted transient class probabilities and rarity alerts. The system supports unsupervised discovery of unknown event types via embedding outlier detection.
Real-Time Alerts Class Probabilities Anomaly Detection ZTF / LSST Ready NASA / ESA Use
